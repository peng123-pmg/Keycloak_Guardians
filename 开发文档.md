## 项目概述

### 产品定位

**Keycloak Guardians** 是一款企业级文件协作与权限管理系统，以 Keycloak 作为统一认证授权中心，提供：

- 🔐 **安全的身份与权限管理**：基于 OIDC/OAuth2 标准协议
- 📁 **个人/团队文件管理**：上传、下载、共享、版本控制
- 👥 **团队协作功能**：小组管理、成员邀请、权限分配
- 🔔 **通知与审计**：操作日志、系统通知、安全审计
- 🚀 **快速集成**：RESTful API，支持多种客户端接入

### 核心价值

- **统一认证**：企业级 SSO 单点登录，一次认证，全平台通行
- **细粒度权限**：基于角色的访问控制（RBAC），支持团队级和资源级权限
- **安全可靠**：HTTPS 全链路加密，敏感数据保护，审计日志追溯
- **易于扩展**：模块化设计，支持对象存储、分布式部署、微服务改造

```bash

# 下载并安装 MySQL 8.0
# https://dev.mysql.com/downloads/mysql/

# 或使用 Docker
docker run -d \
  --name guardians-mysql \
  -e MYSQL_ROOT_PASSWORD=root \
  -e MYSQL_DATABASE=iamkc \
  -e MYSQL_USER=iamkc \
  -e MYSQL_PASSWORD=iamkc \
  -p 3306:3306 \
```

### 2. 配置数据库

使用 root 账号连接 MySQL 并执行：

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS iamkc 
  DEFAULT CHARACTER SET utf8mb4 
  COLLATE utf8mb4_unicode_ci;

-- 创建用户（仅限本地访问）
CREATE USER IF NOT EXISTS 'iamkc'@'localhost' 
  IDENTIFIED BY 'iamkc';

-- 授权
GRANT ALL PRIVILEGES ON iamkc.* TO 'iamkc'@'localhost';
FLUSH PRIVILEGES;
```

### 3. 启动 Keycloak 服务器

#### 3.1 使用 Docker（推荐）

```bash
docker run -d \
  --name guardians-keycloak \
  -e KEYCLOAK_ADMIN=admin \
  -e KEYCLOAK_ADMIN_PASSWORD=admin \
  -p 8080:8080 \
  quay.io/keycloak/keycloak:22.0 start-dev
```

#### 3.2 等待 Keycloak 启动完成

```bash
# 访问 Keycloak 管理控制台
# http://localhost:8080

# 登录账号
# 用户名: admin
# 密码: admin
```

```bash

```

### 2. 导入 Keycloak Realm

1. 访问 Keycloak 管理控制台：http://localhost:8080
2. 使用 `admin` / `admin` 登录
3. 点击左上角 Realm 下拉菜单，选择 **Create Realm**
4. 点击 **Browse...** 按钮，选择文件：
   ```
   keycloak-server/src/main/resources/realm-import.json
   ```
5. 点击 **Create** 按钮完成导入

**Realm 配置说明：**

- **Realm 名称**: `guardians`
- **预置用户**:
  - `admin` / `admin` (角色: admin, user)

## 核心功能

### 1. 身份认证与授权

#### 认证流程

```
用户 → 前端应用 → Keycloak 登录页 
    ↓
  输入用户名密码
    ↓
Keycloak 验证 → 颁发 Access Token
    ↓
前端存储 Token → 调用后端 API (Authorization: Bearer {token})
    ↓
后端验证 Token → 解析用户信息 → 执行业务逻辑
```

#### 角色权限

| 角色 | 权限范围 |
|------|---------|
| **admin** | 系统管理员，拥有所有权限 |
| **user** | 普通用户，可管理个人文件和参与的团队 |
| **user_premium** | 高级用户（可扩展） |

#### Token 配置

- **Access Token 有效期**: 5 分钟
- **Refresh Token 有效期**: 30 分钟
- **Token 刷新**: 前端自动刷新，无感知

### 2. 文件管理

#### 个人文件

- **上传文件**: 支持多文件上传，实时进度显示
- **下载文件**: 流式下载，支持大文件
- **删除文件**: 软删除，进入回收站
- **文件列表**: 分页显示，支持搜索和筛选

#### 团队文件

- **共享文件**: 将个人文件共享到团队
- **团队文件列表**: 查看团队内所有共享文件
- **权限控制**: 只有文件上传者可删除团队文件

#### 文件元数据

```typescript
interface FileInfo {
  id: string;              // 文件ID
  name: string;            // 文件名
  type: string;            // 文件类型: file/image/video/audio/document
  size: number;            // 文件大小(字节)
  uploadTime: string;      // 上传时间(ISO 8601)
  uploader?: string;       // 上传者ID
}
```

### 3. 团队协作

#### 创建团队

- 创建者自动成为团队 OWNER
- 设置团队名称和描述
- 可见性控制（公开/私有）

#### 成员管理

- **邀请成员**: 通过用户名邀请
- **成员角色**:
  - `OWNER`: 创建者，拥有所有权限
  - `MEMBER`: 普通成员，可查看和上传文件
- **移除成员**: 仅 OWNER 可操作

#### 团队权限

```
团队创建 → 仅创建者可以删除团队
文件共享 → 任何成员都可以将文件共享到团队
文件删除 → 仅文件上传者可以删除自己共享的文件
团队文件查看 → 所有成员都可以查看和下载
```

### 4. 回收站

- 删除的文件进入回收站
- 支持文件恢复
- 支持彻底删除
- 可设置自动清理策略（30天）

### 5. 通知中心

- 系统通知：账号变更、配额警告
- 协作通知：成员邀请、文件共享
- 审核通知：权限变更、操作审批
- 支持已读/未读状态

### 6. 安全审计

- 操作日志记录
- 敏感操作审计
- 登录日志追踪
- 异常行为检测

## API 接口文档

### 认证说明

所有 API 请求（除健康检查外）都需要在请求头中携带 Token：

```http
Authorization: Bearer {access_token}
```

### 获取 Token

```http
POST http://localhost:8080/realms/guardians/protocol/openid-connect/token
Content-Type: application/x-www-form-urlencoded

grant_type=password
&client_id=backend-service
&username=admin
&password=admin
```

**响应示例：**
```json
{
  "access_token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expires_in": 300,
  "refresh_expires_in": 1800,
  "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "Bearer"
}
```

### 命名规范

#### 数据库表命名

- 使用小写 + 下划线：`users`, `group_members`, `file_tags`
- 复数形式：`files` 不是 `file`
- 关联表：`table1_table2` 如 `file_tags`

#### API 路径命名

- RESTful 风格
- 使用复数：`/api/files` 不是 `/api/file`
- 资源嵌套：`/api/groups/{id}/members`

#### 变量命名

- Kotlin: camelCase `userId`, `fileList`
- TypeScript: camelCase `userName`, `fileCount`
- 常量: UPPER_SNAKE_CASE `MAX_FILE_SIZE`, `API_BASE_URL`

#### 3. Docker 部署（推荐）

#### 3.1 创建 Docker Compose 配置

```yaml
# docker-compose.yml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: guardians-mysql
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: iamkc
      MYSQL_USER: iamkc
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
    networks:
      - guardians-network

  keycloak:
    image: quay.io/keycloak/keycloak:22.0
    container_name: guardians-keycloak
    command: start-dev
    environment:
      KEYCLOAK_ADMIN: admin
      KEYCLOAK_ADMIN_PASSWORD: ${KEYCLOAK_ADMIN_PASSWORD}
      KC_DB: mysql
      KC_DB_URL: jdbc:mysql://mysql:3306/keycloak
      KC_DB_USERNAME: keycloak
      KC_DB_PASSWORD: ${KEYCLOAK_DB_PASSWORD}
    ports:
      - "8080:8080"
    depends_on:
      - mysql
    networks:
      - guardians-network

  backend:
    build: ./keycloak-server
    container_name: guardians-backend
    environment:
      MYSQL_JDBC_URL: jdbc:mysql://mysql:3306/iamkc
      MYSQL_USERNAME: iamkc
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}
      KEYCLOAK_SERVER_URL: http://keycloak:8080
    ports:
      - "8081:8081"
    depends_on:
      - mysql
      - keycloak
    networks:
      - guardians-network

  frontend:
    image: nginx:alpine
    container_name: guardians-frontend
    volumes:
      - ./Theme/dist:/usr/share/nginx/html
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
    ports:
      - "80:80"
    depends_on:
      - backend
    networks:
      - guardians-network

volumes:
  mysql_data:

networks:
  guardians-network:
    driver: bridge
```

#### 4. 配置反向代理（生产环境）

#### 4.1 Nginx 配置示例

```nginx
# /etc/nginx/conf.d/guardians.conf
upstream backend {
    server localhost:8081;
}

upstream keycloak {
    server localhost:8080;
}

server {
    listen 80;
    server_name guardians.example.com;
    
    # HTTPS 重定向
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name guardians.example.com;
    
    # SSL 证书
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    # 前端静态文件
    location / {
        root /var/www/guardians/dist;
        try_files $uri $uri/ /index.html;
    }
    
    # 后端 API
    location /api/ {
        proxy_pass http://backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    
    # Keycloak
    location /realms/ {
        proxy_pass http://keycloak;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    
    # 文件上传大小限制
    client_max_body_size 100M;
}
```
